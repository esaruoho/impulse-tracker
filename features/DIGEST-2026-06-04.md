> Version note: the fork always ships as **v2.354**, distinguished by a date

Two things happened today, in parallel sessions on the same tree:
1. **Tracker features** — actual changes to how IT behaves when you use it.
2. **Convey** — the self-documenting "report card" machinery around the code.

Grades are honest: **runtime-verified** = you ran it and confirmed;

## A. Tracker features (what you can actually do now)

### 1. Order List — F6 loops, F7 plays from a chosen row  ✅ runtime-verified
On the F11 Order List, **F6** loops the selected order's pattern; **F7** plays
the song from the selected order at the current row. You confirmed both.
- commits `8acb41f`, `5b37353`, `41cbb56`

### 2. Shift-Alt-R = replicate the WHOLE pattern at cursor  · build-verified
Previously Shift-Alt-R was "clear all track views". Now it replicates the whole
pattern at the cursor (Alt-R still does the single track). Two fixes were
needed: the keymap wasn't even reaching the handler (missing the Shift+Alt
"condition 11" entry), and Alt-R / Shift-Alt-R now get their own undo labels.
- commits `5fb263b`, `a52a462`, `3a3b7ff` — (plain Alt-R single-track is hw-verified)

### 3. Shift-F4 drumkit  · build-verified
Shift-F4 now also auto-builds a drumkit on **instrument 01**: every sample is
mapped to a key, on MIDI channel 10. The multitimbral MIDI-in parts moved to
instruments 02–17 to make room.
- commits `f94f63c`, `dee41bd`

### 4. Note-cut toggle — "1" toggles ^^^ off  · build-verified
In the pattern editor, pressing **1** on an existing note-cut (`^^^`) now
removes it (toggles off), instead of only ever placing one.
- commit `81e4819`

### 5. F8 also sends a MIDI Stop out  · build-verified (outbound hw-untested)
**This session's feature.** Pressing **F8** (Stop) now also transmits a single
MIDI Stop (`0FCh`) out the MIDI port, so slaved gear stops too. Governed by a
new **Shift-F1 toggle** ("Send MIDI Stop (FC) on F8", default ON). It
**persists across restarts** (auto-saves to IT.CFG, survives quit/relaunch with
no separate save step). By design there is exactly **one** transmit site (the
F8 keypress) and the inbound MIDI path never transmits — so a MIDI-thru
loopback can't create a Start/Stop feedback storm.
- commits `67cdb60` (transmit + toggle), `222962f` (persistence), `7559a78` (auto-save)

### 6. Shift-Enter bulk-load returns to the directory  · bugfix, build-verified
After Shift-Enter bulk-loads all of a module's samples, the loader now exits the
module back to its file directory instead of leaving you stranded inside it.
- commit `e0b1643`

### 7. Blank song name defaults to a timestamp  · build-verified (parallel session)
If you leave the song name empty, it's defaulted to the creation timestamp
(`YYYY-MM-DD HH:MM`) so untitled tunes are still identifiable.
- commit `87ad1dd`

### Feasibility finding (not a feature): 256/512-row patterns
Investigated and **documented as blocked** — patterns longer than 200 rows are
prevented by the 64 KB real-mode segment limit. Carded so the constraint is on
record rather than re-investigated.
- commit `6208e79`

## B. Convey — the self-documenting machinery (infrastructure)

This is the bigger structural thread of the day. "Convey" is the name given to
the report-card system: every feature carries a **triad** — a `.feature` card
(graded Given/When/Then claims, each cited to a proc+line+commit), a
`.session.md` (the conversation that spawned it), and a RESULT log of commits.
Today that system was named, automated, and made testable:

- **Named "Convey"** + the Tests→Features→auto-index principle (`84ed72e`)
- **STATUS.md is now generated** from the cards' grade tags, not hand-typed
  (`95a4f9e`); every scenario got an honest `@hw-untested` where DOSBox ≠ metal
  (`0250ecf`)
- **Hardware-test sheet** auto-generated (`961d7b4`), plus an interactive
  test TUI `hwtest.py` (`9ec40af`) and a repo-anchored `./test-impulse-tracker`
  launcher (`e63518b`) — the way you walk through unverified features on real
  hardware and flip passes to `@hw-verified`
- **Sessions ledger + single distilled situation** (`a317aa4`), and live Claude
  conversations plugged into the tooling via `gen-sessions.py` (`f1c6f5b`)
- **The "garden" principle** — Convey prunes as well as grows (`ad7fc0e`,
  `0ba2cbf`)
- A 2-day digest card spanning 06-03 → 06-04 (`8ab4ec4`)

## How to verify the still-unconfirmed features on hardware

Run `./test-impulse-tracker` — it walks every fork scenario not yet
`@hw-verified`, records works/failed, and updates the cards. The ones most
worth confirming from today: the **F8 MIDI Stop out** (needs a slaved device or
