**Fix attempted (reverted later):** patched `Glbl_Alt_F12` (`IT_K.ASM:1664`) to call `SetInfoLine2` with `BX = 2100` (~3 s during playback).

## Current MIDI sync feature surface (post-push)

- **Enable flag:** `MIDISyncEnable` byte in `IT_K.ASM:97`. Default = 1 (ON).
- **Toggle UI:** Shift-F1 MIDI screen → MIDI Sync toggle button (calls `Glbl_Alt_F12`).
- **Driver coverage:** any driver with MPU-401 UART receive (`MIDIDRV.DRV` pure MIDI, `SB16DRV.DRV` SB16 onboard MPU-401, etc.). Single interception point: `MIDISend` in `IT_K.ASM`.
- **Tempo update cadence:** every 24 F8 pulses = once per quarter note. No smoothing yet.
- **F8 gating:** governed by `MIDISyncEnable`.
- **Start/Stop gating:** **NOT** governed by the flag — `FA`/`FC` always fire. If you want the toggle to fully disarm external sync, add the flag-check at the top of `MIDISendRTStart` and `MIDISendRTStop`.

## Open user-facing issues at end of session

### A. "Pattern Size Mismatch — Fix: Remove corrupted data" warning
Esa loaded a regular song with the new `IT.EXE` (post-`7163709`) and got the warning. Source: `IT_PE.ASM:7913` `DecodePattern` — fires when stored decoded byte count ≠ expected `DecodePatternBytes`.

- None of this session's commits touch `IT_PE.ASM`, `IT_M.ASM`, or pattern-decode logic.
- Data layout shift from inserting `DirectoryPickerActive DB 0` between `QuickSaveBackup` and `QuickSaveNoFolderMsg` is symbol-resolved by the linker (verified all references are by `Offset Symbol`, not byte arithmetic).
- **Verification step pending Esa:** load the same file with the older `IT2354-esaruoho-ad5d840/IT.EXE` (snapshot at `ad5d840`, pre-session). If the warning fires there too → pre-existing, not a regression. If only the new build warns → real regression to investigate.

### B. MIDI Start (`0xFA`) appears not to be arriving
Esa: MIDI sync ON, "definitely receiving some sort of information" but playback doesn't start.

- The existing `K_ShowMIDIInput` info-line display only renders **channel** messages. System Real-Time bytes (`F8`/`FA`/`FC`) are intercepted earlier in `MIDISend` (`IT_K.ASM:1813-1822`) and never reach the display ring. Esa seeing "data" doesn't prove `0xFA` is arriving.
- `MIDISendRTStart` (`IT_K.ASM:1824`) calls `Music_KBPlaySong` unconditionally. So if `0xFA` arrives, IT WILL start.
- **Most likely cause:** Esa's DAW is sending Clock but not Start/Stop/Transport. Many DAWs split these into two settings. Check for "Send MIDI Start/Stop", "Transport sync", or "MIDI Machine Control" in DAW MIDI prefs.
- The MIDI monitor (next-session priority A) will definitively answer this.

### B. Sample / Instrument / Quicksave directory pickers
The Module Directory MVP shipped (`8ca7078`); the other three rows still inline-edit only.

Architecture: each non-Module picker needs a `SongDirectory` ↔ target backup/restore wrap because F9 mutates `SongDirectory`, not `SampleDirectory`/`InstrumentDirectory`/`QuickSaveDirectory`.

Skeleton per row (Sample shown):
```asm
Proc D_PickSampleDir Far
    Push CS / Pop DS / Assume DS:Disk
    ; backup current SongDirectory
    Mov  SI, Offset SongDirectory
    Mov  DI, Offset DirectoryPickerOrigDir
    Mov  CX, 70
    Rep  MovsB
    ; seed SongDirectory from SampleDirectory so F9 starts there
    Mov  SI, Offset SampleDirectory
    Mov  DI, Offset SongDirectory
    Mov  CX, 70
    Rep  MovsB
    Mov  DirectoryPickerActive, 1
    Mov  DirectoryPickerMode, 1     ; 1 = Sample, 2 = Instrument, 3 = QuickSave
    Jmp  Glbl_F9
EndP
```

Extend `D_PickerEsc` to dispatch on `DirectoryPickerMode`:
- 0 (Module): no copy; SongDirectory IS target.
- 1/2/3: copy SongDirectory → SampleDirectory / InstrumentDirectory / QuickSaveDirectory, restore SongDirectory from backup.

Add 2 state bytes:
```
DirectoryPickerMode      DB      0
DirectoryPickerOrigDir   DB      70 Dup (0)
```

Wire 3 more `DD 0` callbacks in `IT_OBJ1.ASM` (SampleDirectoryInput line 5560, InstrumentDirectoryInput line 5567, QuickSaveDirectoryInput line 5574).

~80 lines TASM, one commit.
